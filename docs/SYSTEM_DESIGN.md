## 二、核心模块设计

### 2.1 OKX 接口层 (Exchange Gateway)

**职责：** 与 OKX 交易所的唯一通信层

| 功能 | 接口类型 | OKX API |
|------|----------|---------|
| 行情订阅（Ticker/深度/K线） | WebSocket | 公共频道 |
| 账户余额/持仓监控 | WebSocket | 私有频道 |
| 订单状态推送 | WebSocket | 私有频道 |
| 下单/撤单/改单 | WebSocket（优先）/ REST | 交易接口 |
| 历史K线/成交获取 | REST | 行情接口 |
| 账户配置/杠杆设置 | REST | 账户接口 |

**关键设计：**
- WebSocket 优先：下单用 WS 比 REST 快 ~50ms
- 自动重连机制：断线后 1s/2s/4s/8s 指数退避重连
- 心跳保活：每 25s 发送 ping
- 限速管理：内置令牌桶，遵守 OKX 限速规则（下单 60次/2s，撤单 60次/2s）
- 签名模块：HMAC SHA256 签名，支持 API Key + Secret + Passphrase

### 2.2 数据层 (Data Pipeline)

**实时数据流：**
```
OKX WebSocket → 数据解析 → 数据标准化 → 分发给策略引擎
                                      → 存储到本地（SQLite/TimescaleDB）
```

**采集的数据：**
- **Ticker：** 最新成交价、24h涨跌幅、成交量
- **深度数据（Order Book）：** 买卖5/20/50档
- **K线数据：** 1m / 5m / 15m / 1h / 4h / 1d
- **成交流（Trades）：** 逐笔成交
- **资金费率：** 合约资金费率（每8小时）
- **标记价格 & 指数价格：** 用于合约风控

### 2.3 策略引擎 (Strategy Engine)

**架构：策略插件化，每个策略独立运行**

```python
class BaseStrategy:
    def on_tick(self, ticker)      # 每次 tick 触发
    def on_kline(self, kline)      # K线闭合触发
    def on_depth(self, orderbook)  # 深度更新触发
    def on_trade(self, trade)      # 成交推送触发
    def on_order(self, order)      # 订单状态变化触发
    def on_position(self, pos)     # 持仓变化触发
    def generate_signal(self) → Signal  # 生成交易信号
```

**内置策略（第一阶段）：**

#### 策略1：趋势追踪（Trend Following）
- 多时间框架分析：4h 定方向，1h 找入场，15m 精确触发
- 指标组合：EMA(20/50/200) + MACD + RSI + ATR
- 入场：趋势确认 + 回调到均线支撑
- 出场：移动止盈 + ATR 动态止损

#### 策略2：均值回归（Mean Reversion）
- 布林带 + RSI 超买超卖
- 适用于震荡行情
- 严格止损：偏离 2σ 以上强制止损

#### 策略3：动量突破（Momentum Breakout）
- 成交量确认的价格突破
- 订单簿厚度分析辅助判断真假突破
- 突破后追踪止盈

#### 策略4：资金费率套利（Funding Rate Arbitrage）
- 现货 + 合约对冲
- 费率 > 阈值时开仓吃费率
- 极低风险，适合大资金

#### 策略5：网格交易（Grid Trading）
- 利用 OKX 原生网格 API 或自建网格
- 震荡市自动高抛低吸
- 智能网格间距（基于 ATR）

### 2.4 风控引擎 (Risk Management) — 最核心模块

**铁律：活着比赚钱重要**

#### 单笔风控
| 规则 | 参数 |
|------|------|
| 单笔最大亏损 | 账户净值的 1-2% |
| 强制止损 | 每笔交易必须设置止损，无止损不允许开仓 |
| 止损方式 | ATR 动态止损 / 固定百分比止损 / 支撑阻力止损 |
| 盈亏比要求 | 最低 1.5:1，目标 2:1 以上 |
| 滑点保护 | 限价单为主，市价单预留滑点 |

#### 账户级风控
| 规则 | 参数 |
|------|------|
| 最大持仓数 | 同时不超过 N 个（可配置，默认5） |
| 总仓位暴露 | 不超过账户净值的 X%（可配置） |
| 单日最大亏损 | 账户净值的 5%，触发暂停交易 |
| 连续亏损 | 连续 3 笔亏损，降低仓位 50% |
| 回撤保护 | 最大回撤 10% 触发全部平仓并暂停 |
| 杠杆限制 | 合约杠杆上限（可配置，建议 ≤10x） |

#### 合约专项风控
| 规则 | 说明 |
|------|------|
| 爆仓预警 | 保证金率 < 150% 时减仓 |
| 资金费率监控 | 持仓方向与费率方向一致时预警 |
| 双向持仓管理 | 支持同时开多开空（对冲模式） |
| 强平保护 | 接近强平价时自动减仓 |

#### 市场异常检测
- 闪崩检测：价格 1 分钟内波动 > 3% → 暂停开仓
- 流动性检测：买卖价差 > 阈值 → 拒绝交易
- API 异常：连续下单失败 → 暂停策略

### 2.5 仓位管理 (Position Sizing)

**凯利公式 + 固定比例结合：**

```
仓位 = min(Kelly仓位, 固定比例仓位, 最大仓位限制)

Kelly = (胜率 × 盈亏比 - (1 - 胜率)) / 盈亏比
固定比例 = 账户净值 × 风险比例 / (入场价 - 止损价)
```

**动态调仓：**
- 盈利时：逐步加仓（金字塔加仓）
- 亏损时：缩小仓位
- 高波动时：减少仓位
- 低波动时：适当增加仓位

### 2.6 回测 & 模拟 (Backtest & Paper Trading)

**三阶段验证：**

```
历史回测 → 模拟盘验证 → 小资金实盘 → 正式实盘
```

- **回测引擎：** 支持滑点模拟、手续费模拟、资金费率模拟
- **模拟盘：** 接入 OKX 模拟盘 API（`https://www.okx.com` 模拟盘环境）
- **绩效指标：** 夏普比率、最大回撤、胜率、盈亏比、Calmar比率

## 三、交易执行流程

```
1. 数据采集层接收实时行情
    ↓
2. 数据分发到各策略引擎
    ↓
3. 策略引擎产生交易信号 (Signal)
    ↓
4. 信号进入风控引擎审核
    ├── 通过 → 5. 计算仓位大小
    └── 拒绝 → 记录原因，不执行
         ↓
5. 仓位管理计算下单量
    ↓
6. 订单管理器执行下单
    ├── 限价单优先（减少滑点）
    ├── 同时挂止损单
    └── 设置止盈目标
    ↓
7. 订单状态监控（WebSocket 推送）
    ├── 成交 → 更新持仓、记录交易
    ├── 部分成交 → 等待 / 调整
    └── 超时未成交 → 撤单 / 改价
    ↓
8. 持仓管理
    ├── 动态调整止损（移动止盈）
    ├── 监控盈亏比
    └── 触发出场条件 → 平仓
    ↓
9. 交易记录 & 绩效统计
```

## 六、合约交易设计

### 6.1 支持的合约类型
- **USDT 保证金永续合约** (如 BTC-USDT-SWAP)
- **币本位永续合约** (如 BTC-USD-SWAP)
- **交割合约** (如 BTC-USD-240329)

### 6.2 多空操作

| 操作 | 参数 |
|------|------|
| 开多 | `side=buy, posSide=long` |
| 平多 | `side=sell, posSide=long` |
| 开空 | `side=sell, posSide=short` |
| 平空 | `side=buy, posSide=short` |

### 6.3 保证金模式
- **全仓 (cross)：** 所有仓位共享保证金，资金利用率高
- **逐仓 (isolated)：** 每个仓位独立保证金，风险隔离
- 建议：默认逐仓，隔离单笔交易风险

## 七、"稳定赚钱"的核心逻辑

### 7.1 不是每笔都赚，而是整体赚

```
期望收益 = 胜率 × 平均盈利 - (1-胜率) × 平均亏损 > 0
```

**两种赚钱路径：**
- **高胜率 + 小盈亏比：** 均值回归、网格交易（胜率 60%+，盈亏比 1:1）
- **低胜率 + 大盈亏比：** 趋势追踪（胜率 35-45%，盈亏比 2:1+）

### 7.2 多策略组合

- 趋势市用趋势策略
- 震荡市用网格/均值回归
- 费率套利持续产生现金流
- **市场状态分类器** 自动切换策略权重

### 7.3 复利增长

- 盈利后适度加大仓位
- 亏损后缩小仓位
- 定期提取利润（如每月提取 30% 利润）
